# Ópticas Car & Era — Changelog histórico

> **Este archivo NO se carga automáticamente en el contexto de Claude Code** (no es un `CLAUDE.md`).
> Sirve como referencia profunda y consultable para investigar contexto histórico.

## Por qué existe este archivo

`CLAUDE.md` se inyecta completo en cada turno de conversación. Tener ~120 entradas de changelog ahí consumía ~55k tokens de contexto fijo en cada respuesta. Movido a este archivo separado el 2026-04-15 para reducir overhead sin perder historia.

## Cuándo consultarlo

- Buscar contexto histórico de un módulo, feature o bug pasado (ej: por qué se diseñó así, qué intentos previos hubo)
- Revisar **lecciones aprendidas** de incidentes (buscar `LECCIÓN`)
- Ver evolución de un sistema completo (Magnolia, Clari, Clip, RH, Caja, etc.)
- Investigar por qué se tomó una decisión técnica
- Diagnosticar bugs recurrentes (revisar versiones que tocaron el mismo área)

## Cómo buscar

Usar **Grep** con palabra clave:
- `Grep "magnolia" CHANGELOG.md` — todo lo relacionado con Magnolia
- `Grep "LECCIÓN v" CHANGELOG.md -A 3` — todas las lecciones aprendidas
- `Grep "clip-webhook" CHANGELOG.md` — historia del webhook de Clip
- `Grep "^Cambios v25" CHANGELOG.md` — solo cambios de la familia v250-v259

## Lo que NO está en este archivo

Las **reglas operacionales activas** y **decisiones de arquitectura vigentes** viven en `CLAUDE.md` (secciones REGLAS QUE NUNCA SE ROMPEN, módulos, integraciones). Este archivo es contexto histórico, no operación diaria. Si una regla se descubre/refuerza con un cambio, se agrega también a `CLAUDE.md` para que esté siempre cargada.
